## Notable patterns

**Protocol bug (tc02, tc06):** The agent produces a correct, well-reasoned
text answer but never calls `finish()`. The agent loop interprets this as
`stopped_reason=max_steps` (agent.py lines ~187-193). This is a real bug —
the `stopped_reason_is: finish` assertion correctly catches it. The LLM judge
independently evaluates answer quality and scores these refusals highly
(0.82–0.94), separating protocol compliance from answer quality.

**Flakiness evidence:** tc06's judge score range is 0.00–1.00 across all
saved runs. The 0.00 outlier comes from a concurrent run where the agent hit
a rate limit mid-loop (`stopped_reason=error`), produced no answer, and the
judge correctly scored it 0.00. Run `python -m eval.judge_validate --case
tc06_out_of_corpus` to inspect rationales.
